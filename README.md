В 1 задании создавался генератор файлов по определенным правилам присваивания имен папок и файлов, количество создаваемых папок и файлов, а также вес файлов передавалось через параметры при запуске скрипта. Путь от корня и вес файлов при запуске скрипта сохраняются в лог файле. Во 2 задании создавался генератор файлов с целью засорить память системы до определенного уровня. В 3 задании создавался скрипт по очищению системы 3 способами: по лог файлу, по времени создания, по маске имени. В 4-6 заданиях создавался генератор логов nginx в combined формате и скрипт для мониторинга логов (через awk) с несколькими режимами сортировки логов. Вывод логов в первом случае сохранялся в текстовый файл, во втором - визуализировался с помощью GoAccess через веб-интерфейс. 7-8 задания были посвящены системному мониторингу с помощью таких инструментов как Prometheus (для сбора метрик от Node Exporter) и Grafana (для создания дашбордов с диаграммами, визуализирующими собранные данные). Собирались основные метрики системы (ЦПУ, оперативная память, свободное место на диске и кол-во операций ввода/вывода) и наблюдались изменения метрик при: запуске генератора файлов засорения системы; отрабатывании утилиты stress; нагрузки сети с помощью утилиты iperf3 (для этого создавалась вторая виртуальная машина в подсети первой). В 9 задании создавался кастомный node_exporter путем создания конфигурации nginx для отдачи метрик prometheus и проводились те же тесты как в заданиях 7-8.
